# ШАБЛОНЫ ПРОМПТОВ ДЛЯ VS CODE

## ШАБЛОН 1: ДОБАВИТЬ НОВЫЙ ЭКРАН

```
# ЗАДАЧА: Добавить экран [НАЗВАНИЕ]
# ПУТЬ: /Users/slikair369/Desktop/mlm-community/

## ЧТО СОЗДАТЬ

1. templates/[name].html — разметка экрана (<500 строк)
2. js/[name].js — логика + init[Name]() + window.init[Name] (<500 строк)
3. css/[name].css — стили (<800 строк)
4. Обновить router.js — добавить в TEMPLATES + вызов init в ensureTemplate
5. Обновить index.html — подключить js/[name].js и css/[name].css

## ДИЗАЙН
[описание секций экрана, ссылки на макет]

## ЗАПРЕТЫ
- НЕ менять существующие файлы (кроме router.js и index.html)
- НЕ превышать лимиты строк
- НЕ добавлять библиотеки

## ПРОВЕРКИ
- wc -l templates/[name].html js/[name].js css/[name].css
- node --check js/[name].js
- grep "window.init[Name]" js/[name].js
```

## ШАБЛОН 2: ИЗМЕНИТЬ СУЩЕСТВУЮЩИЙ ФАЙЛ

```
# ЗАДАЧА: Изменить [файл]
# ПУТЬ: /Users/slikair369/Desktop/mlm-community/

## РЕЗЕРВНАЯ КОПИЯ
cp [файл] [файл].backup.[описание]

## ЧТО МЕНЯТЬ
Файл: [путь]
Строки: [номера]
Изменение: [описание]

## НЕ ТРОГАТЬ
- [список файлов и функций которые нельзя менять]

## ПРОВЕРКИ
- wc -l [файл] (< лимита)
- node --check [файл] (если JS)
- diff [файл].backup [файл]
```

## ШАБЛОН 3: ИСПРАВИТЬ БАГ

```
# ЗАДАЧА: Исправить [описание бага]
# ПУТЬ: /Users/slikair369/Desktop/mlm-community/

## СИМПТОМЫ
[что не работает]

## ОЖИДАЕМОЕ ПОВЕДЕНИЕ
[как должно работать]

## ФАЙЛЫ ДЛЯ ИЗМЕНЕНИЯ
Только: [список файлов]

## ЗАПРЕТЫ
- НЕ менять другие файлы
- НЕ добавлять новые зависимости

## ПРОВЕРКИ
- node --check [файл]
- diff [файл].backup [файл]
```

## ШАБЛОН 4: РЕФАКТОРИНГ

```
# ЗАДАЧА: Рефакторинг [файл/модуль]
# ПУТЬ: /Users/slikair369/Desktop/mlm-community/

## РЕЗЕРВНАЯ КОПИЯ
cp [файл] [файл].backup.refactor

## ЦЕЛЬ
[что улучшаем: вынести в функцию, разбить файл, убрать дубли]

## ПРАВИЛА
- REPLACE, не дублировать
- Размер < лимита после рефакторинга
- Логика должна остаться идентичной

## НЕ ТРОГАТЬ
[что не менять]

## ПРОВЕРКИ
- wc -l [файл]
- node --check [файл]
- diff [файл].backup.refactor [файл]
```

## ВАЖНО: ЧИТАТЬ ДОКУМЕНТЫ ПЕРЕД РАБОТОЙ

Каждый промпт должен начинаться с этапа 0 — чтение правил проекта.

## ПОЛНЫЙ ШАБЛОН (ИСПОЛЬЗОВАТЬ ВСЕГДА)

```
# ═══════════════════════════════════════════════════
# ПРОЕКТ: MLM COMMUNITY — [НАЗВАНИЕ ЗАДАЧИ]
# ПУТЬ: /Users/slikair369/Desktop/mlm-community/
# ЗАДАЧА: [Описание]
# ═══════════════════════════════════════════════════

## ЗАДАЧА
[Детали]

## КОНТЕКСТ
[Зачем]

# ЭТАП 0: ПРОЧИТАТЬ ПРАВИЛА
cat ARCHITECTURE.md
cat WORKFLOW.md

# ЭТАП 1: РЕЗЕРВНЫЕ КОПИИ (ОБЯЗАТЕЛЬНО!)
cp файл1 файл1.backup.[описание]
cp файл2 файл2.backup.[описание]

# ЭТАП 2: РАБОТА
[выполнение задачи]

# ЭТАП 3: ПРОВЕРКИ (ОБЯЗАТЕЛЬНО!)
wc -l файлы
node --check файл.js
grep "console.log\|TODO\|FIXME\|debugger" файл
diff файл.backup файл

# ЭТАП 4: ОТЧЁТ
РЕЗЕРВНЫЕ КОПИИ: файл.backup
РАЗМЕР: было XXX -> стало XXX (<лимит)
ПРОВЕРКИ: размер, синтаксис, мёртвый код, diff
СТАТУС: Готово
```
